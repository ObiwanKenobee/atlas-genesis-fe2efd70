# Support

## How to Get Help

### Documentation

Start here — most questions are answered in the docs:

- **[README.md](README.md)** — Platform overview and quick start
- **[ARCHITECTURE.md](ARCHITECTURE.md)** — Technical architecture
- **[docs/](docs/)** — Full documentation including API reference, deployment guides, and ADRs
- **[openapi.yaml](openapi.yaml)** — Complete API specification

### GitHub Discussions

For questions, ideas, and community conversation:

**[github.com/AtlasSanctum/atlas-sanctum/discussions](https://github.com/AtlasSanctum/atlas-sanctum/discussions)**

Categories:
- **Q&A** — Ask questions, get answers
- **Ideas** — Propose features and improvements
- **Show and Tell** — Share what you've built with Atlas Sanctum
- **Roadmap** — Discuss and influence the roadmap

### GitHub Issues

For confirmed bugs and feature requests:

**[github.com/AtlasSanctum/atlas-sanctum/issues](https://github.com/AtlasSanctum/atlas-sanctum/issues)**

Use the issue templates — they help us help you faster.

### Email Support

For enterprise support, partnerships, and sensitive matters:

| Topic | Contact |
|-------|---------|
| General | hello@atlassanctum.org |
| Enterprise support | enterprise@atlassanctum.org |
| Security vulnerabilities | security@atlassanctum.org |
| Governance | governance@atlassanctum.org |
| Press & media | press@atlassanctum.org |

### Security Issues

**Never report security vulnerabilities in public issues.**

See [SECURITY.md](SECURITY.md) for the responsible disclosure process.

## Response Times

| Channel | Expected Response |
|---------|------------------|
| GitHub Discussions | 1–3 business days |
| GitHub Issues (bugs) | 3–5 business days |
| Enterprise email | 1 business day |
| Security reports | 48 hours |
